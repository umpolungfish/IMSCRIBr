Branch polarity rule: AREV anchors F-branch, AFWD anchors T-branch

In imscr_wiring(), branch assignment inside FSPLIT/FFUSE pairs previously
used only EVALT/EVALF as anchors. When neither was present, all interior
nodes fell to the T-branch, misplacing AREV (parity flip / reverse morphism)
on the affirmative arm.

Fix: secondary anchor lookup after the primary EVALT/EVALF check.
  - If EVALF absent: AREV (pol/𐑗, parity flip) anchors the F-branch.
  - If EVALT absent: AFWD (rel/𐑾, forward morphism) anchors the T-branch.

Secondary anchors activate only when the corresponding primary anchor
(EVALT/EVALF) is absent; existing behavior is fully preserved for all
arrangements that already contain EVALT or EVALF.

Verified on ER=EPR sequence: AFWD+EVALT → T-branch, AREV+EVALT → F-branch.
All 4 canonical classes with matched FSPLIT/FFUSE pairs still validate.
