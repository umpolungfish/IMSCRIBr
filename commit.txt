proof_scaffold.py: cap the Lean identifier stem so free-text class names are usable

emit_scaffold() derived every Lean identifier (s0..s12, l0..l12, protocol,
true_arm, false_arm, tier, frobenius) from the full class name with no length
cap. For a free-text ob3ect class (often a whole sentence) this produced 300-char
identifiers repeated dozens of times, i.e. unusable Lean.

Fix: after slugging, if the stem exceeds 40 chars, truncate to a word-boundary
stem and append a 6-char sha1 suffix so distinct long names cannot collide. The
full class name is unchanged in the header ("-- Class: ..."). Short/canonical
names are untouched.

Verified: the Navier-Stokes ob3ect scaffold now uses idents like
the_non_vacuous_structural_type_of_408149_s0 with the full sentence still in the
header comment.

symbolic_diagram.py — see prior commit 256aa43.)
