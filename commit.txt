Newton Lift: math-preservation / rigor correction (2026-06-15)

Closed a significant design bug reported directly by the user: the pipeline
was stripping ALL mathematics out of dissolved prose, for every domain,
including ones (physics, mathematics, chemistry) whose findings should be
expressed as real conventional equations and properly defined mathematical
objects, not narrative paraphrase. This was NOT the intended behavior --
only the grammar's own private vocabulary (Shavian glyphs, primitive names,
tier labels, crystal/catalog jargon) was ever supposed to be banned.

Root causes (two, both fixed):

1. GRAMMAR_PATTERNS banned the literal text `\circ \delta`, which silently
   forbade writing the real, standard Frobenius-special-algebra equation
   `mu . delta = id` -- legitimate, pre-existing category-theory/TQFT
   terminology, not invented by this grammar. Pattern removed.

2. DISSOLVE_SYSTEM_PROMPT taught the model, via a literal worked example,
   to flatten a quantitative finding (d(X,Y)=0.000) into a purely
   qualitative sentence with zero equation content -- correct only for
   domains with no native formal apparatus (ecology, civilization), wrong
   as a universal rule. Rewrote the prompt and DISSOLVE_USER_TEMPLATE to
   mandate real domain-conventional formalism (define the actual objects --
   Hilbert space, algebra, manifold, tensor -- write the actual equation,
   carry it to standard closed form) wherever the target domain supports it.

Judge-side enforcement added so this stays fixed going forward:
- verify_semantics() gained a third check, RIGOR, alongside the existing
  FIDELITY and ELEVATION checks, in the same single judge call.
- SemanticVerificationResult gained `rigor_issues: List[Tuple[idx, problem]]`;
  a non-empty rigor_issues now fails the overall gate exactly like a
  fidelity or elevation failure.
- JUDGE_USER_TEMPLATE now passes TARGET DOMAIN so the judge can apply
  domain-appropriate rigor expectations.
- lift()'s retry-feedback loop gained a RIGOR ISSUE feedback branch naming
  the under-formalized findings on retry.

Secondary bug found and fixed while wiring the above: the existing
grammar-violation retry feedback told the model to rewrite with "no
symbols, no formal notation of any kind" on any private-vocabulary leak --
which would have silently re-imposed the no-math rule on retry regardless
of the rigor fix. Narrowed that feedback to target only the grammar's
private vocabulary, explicit that real domain notation is required.

Verified end-to-end with the rigor gate live:
- qg_bridge_findings.json (physics) -> passed on first attempt, producing
  genuine formalism: a Frobenius algebra object with the real relation
  (m⊗id)∘(id⊗Δ)=Δ∘m=(id⊗m)∘(Δ⊗id), the Einstein equation as an operator
  identity G_{μν}+Λg_{μν}=(8πG/c^4)<T_{μν}>, Tomita-Takesaki references,
  type II_1 factor algebras, partition-function functional equations,
  braid groups -- not narrative paraphrase of any of these.
- test_findings.json (ecology/civilization, no native formal apparatus) ->
  also passed, confirming the rigor gate does not falsely fire when prose
  is the domain-appropriate register.
- python3 -m py_compile newton_lift.py -- clean.
- Removed one duplicate ledger entry that my own verification re-run of
  qg_bridge_findings.json appended a second time under the same source
  title (test artifact, not a real new publication); pathway_ledger.json
  restored to its prior physics/ecology entries plus none added by this
  session's testing.

Files touched: newton_lift.py, NEWTON_LIFT_DESIGN.md (§1.4 correction note,
