symbolic_diagram.py: restore black-and-white pen diagram output

pen_mode was accepted by render_wiring_svg_v3 but ignored in the body — the actual
B&W renderer had been lost (never committed), so every diagram came out in color
regardless of the flag.

Restored pen output as the dark->light theme flip it fundamentally is:
  - _to_pen_bw(svg_text): maps every colour to grayscale-by-luminance then inverts
    (0.299R+0.587G+0.114B, then 255-lum). Dark background -> near-white, light
    foreground marks -> near-black, saturated fills -> neutral grays. Alpha suffixes
    preserved; handles #rgb / #rgba / #rrggbb / #rrggbbaa.
  - SVGBuilder gains a pen_mode flag and a build() method; save() writes build(),
    which applies _to_pen_bw when pen_mode is set.
  - render_wiring_svg_v3 sets svg.pen_mode = pen_mode.
  - added `import re`.

Verified: for a 12-opcode artifact the pen SVG has 0 saturated colours (pure
grayscale) vs 25 in the color SVG, white background (#f4f4f4), and rendered to PNG
it is a clean, legible black-and-white pen diagram. Paired with ob3ect auto.py,
which now emits both <slug>_diagram.svg and <slug>_diagram_pen.svg.
